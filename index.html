Connect to InfoTrack MCP

Getting Your Client Credentials

1

Access the Partners Portal

Navigate to partners.infotrack.com and log in to your account.

2

Generate Client Credentials

Create a new application and generate your ClientId and ClientSecret. Keep these secure.

3

OAuth 2.0 Authentication

Your MCP client will handle the OAuth 2.0 Authorization Code flow automatically, authenticating via auth.infotrack.com.

üîê
OAuth 2.0 Details

The MCP client handles the Authorization Code flow automatically. Token endpoint: http://auth.infotrack.com/oauth/token with scope api.

Configure Your MCP Client

Add the following to your ~/.cursor/mcp.json file:

mcp.json 
{
  "mcpServers": {
    "infotrack": {
      "type": "http",
      "url": "https://search.infotrack.com/services/mcp-gateway/servers/infotrack-us/mcp",
      "authorization": {
        "type": "oauth2",
        "clientId": "<your-client-id>",
        "clientSecret": "<your-client-secret>",
        "tokenUrl": "http://auth.infotrack.com/oauth/token",
        "scope": "api"
      }
    }
  }
}

How It Works

Search & Invoke Pattern

InfoTrack MCP uses a dynamic tool discovery pattern. Instead of calling tools directly, you describe what you want to do and our gateway finds the appropriate tools for you.

üîç

search

Describe what you want to do in natural language. The gateway finds the appropriate tools and returns the JSON schema needed to execute them.

‚ö°

invoke

Execute a discovered tool by passing the tool name and the data matching the returned schema. The gateway handles the execution.

How the Pattern Works

1

Search for Tools

Call search with a natural language query describing your intent.

Search Request 
{
  "query": "I want to file documents into a court"
}
2

Receive Tool Schema

The gateway returns the discovered tool with its description and JSON schema.

Search Response 
{
  "tool": "start_filing",
  "description": "Start a court filing with attached PDFs.",
  "schema": {
    "type": "array",
    "items": { "type": "string", "format": "uri" }
  }
}
3

Invoke the Tool

Call invoke with the tool name and data matching the schema.

Invoke Request 
{
  "tool": "start_filing",
  "data": [
    "https://example.com/complaint.pdf",
    "https://example.com/exhibit-a.pdf"
  ]
}
üí°
Why Search & Invoke?

This pattern allows InfoTrack to continuously expand available services without requiring client updates. Your AI agent always has access to the latest tools through natural language discovery.
